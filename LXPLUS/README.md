# Geant4 Installation Guide for CERN LXPLUS

A complete step-by-step guide for setting up the Geant4 development environment, compiling applications, running simulations, and performing basic analysis on CERN LXPLUS.

---

This guide covers:

# 1. Connect to CERN LXPLUS

## Purpose

Connect to the CERN LXPLUS cluster using SSH in order to prepare the Geant4 working environment.

## Prerequisites

Before connecting, make sure that:

- You have a CERN account.
- Your CERN account has LXPLUS access enabled.
- SSH is installed on your computer.

## Connection

Open a terminal and connect using:

```bash
ssh username@lxplus.cern.ch
```

Replace **username** with your CERN username.

## Verification

A successful login displays the LXPLUS welcome message and your home directory.

---

# 2. Setup Geant4 Environment

## Purpose

Configure the CERN LXPLUS environment by loading the required software packages and environment variables required for Geant4.

## Setup

Load the Geant4 software environment before compiling or running any applications.

```bash
source /cvmfs/sft.cern.ch/lcg/views/LCG_107/x86_64-el9-gcc13-opt/setup.sh
```

> **Important:** Run this command every time you start a new LXPLUS session.

## Verification

Verify that the Geant4 environment has been loaded successfully.

```bash
geant4-config --version
```

Expected output:

```text
11.x.x
```---

# 3. Create a Working Directory

## Purpose

Create a dedicated workspace for Geant4 applications.

## Create Directory

```bash
mkdir Geant4_Work
cd Geant4_Work
```

## Verification

```bash
pwd
```

Expected output:

```text
/home/username/Geant4_Work
```---

# 4. Compile Applications

## Purpose

Compile the Geant4 application using CMake and Make.

## Configure

```bash
mkdir build
cd build

cmake ..
```

## Build

```bash
make -j4
```

> **Note:** The `-j4` option compiles using four CPU cores. Adjust this value according to the available resources.

## Verification

Successful compilation generates the executable without any errors.---

# 5. Run Simulations

## Purpose

Run the compiled Geant4 application.

## Execute

```bash
./exampleB1
```

or

```bash
./yourApplication
```

## Verification

The application starts successfully and produces the expected output files.---

# 7. Troubleshooting

## Common Issues

### Geant4 environment is not loaded

Run:

```bash
source /cvmfs/sft.cern.ch/lcg/views/LCG_107/x86_64-el9-gcc13-opt/setup.sh
```

---

### Command not found

Check that the environment has been loaded correctly.

```bash
geant4-config --version
```

---

### Build errors

Delete the build directory and configure the project again.

```bash
rm -rf build

mkdir build
cd build

cmake ..
make
```
